## Part 3. Mini Web Server

> Russian version: [Part3_ru.md](../ru/Part3_ru.md)

### 3.1. Creating a FastCGI Application

Create a file named `hello.c` that implements a simple FastCGI application.

![hello.c](../../images/part3/1.png)

The program accepts requests from a web server and returns an HTML page containing:

```html
<h1>Hello World!</h1>
```

> Application source code: [src/final/hello.c](../../src/final/hello.c)

---

### 3.2. Running the FastCGI Server

Create a new container for the task:

```bash
docker run -d -p 81:81 nginx
```

![docker run](../../images/part3/2.png)

Install the required packages:

```bash
docker exec -it <container_id> apt update
docker exec -it <container_id> apt upgrade -y
docker exec -it <container_id> apt install -y \
    gcc \
    libfcgi-dev \
    libfcgi0ldbl \
    spawn-fcgi
```

Copy the source code into the container, compile the application, and start it using `spawn-fcgi` on port `8080`.

```bash
docker cp hello.c <container_id>:/hello.c
docker exec <container_id> gcc hello.c -o hello.fcgi -lfcgi
docker exec <container_id> spawn-fcgi -p 8080 hello.fcgi
```

![spawn-fcgi](../../images/part3/3.png)

---

### 3.3. Configuring Nginx

Create a custom `nginx.conf` file.

![nginx.conf](../../images/part3/4.png)

The purpose of this configuration is to accept requests on port `81` and forward them to the FastCGI application running on port `8080`.

Key directives:

* `fastcgi_pass 127.0.0.1:8080;` — forwards requests to the FastCGI server;
* `include fastcgi_params;` — includes the standard FastCGI parameters.

> Configuration file used in this part: [src/history/Part3/nginx.conf](../../src/history/Part3/nginx.conf)

Copy the configuration into the container and restart Nginx.

![nginx.conf](../../images/part3/5.png)

---

### 3.4. Verifying the Application

Open the following page in a browser:

```text
http://localhost:81
```

![localhost:81](../../images/part3/6.png)

The page displays the response generated by the FastCGI application.

---

## Summary

A FastCGI application written in C was created and configured to run on port `8080`. An Nginx configuration was added to forward requests to the application.
