### Сканируем образ из предыдущего задания через `dockle [image_id|repository]`

Копируем из папки с предыдущим заданием (*Part4*) все необходимые файлы для компиляции образа и компилируем новый образ в текущей папке. После чего заархивируем образ *hello* в файле *hello.tar* (почему-то **dockle** отказывается сканировать сам образ, как его ни вписывай).

![docker save](../../images/part5/1.png)

Теперь можно просканировать полученный архив.

![dockle](../../images/part5/2.png)

Как видим, у нас есть **ошибки (*FATAL*)**, **предупреждения (*WARN*)** и **рекомендации (*INFO*)**:

  - `FATAL - DKL-DI-0005: Clear apt-get caches`:
      
      Не очищается кэш `apt-get`, что может увеличить размер образа.
      
      Рекомендуется после команд `apt update` и `apt install` добавьте команду `rm -rf /var/lib/apt/lists`, чтобы удалить временные файлы, созданные *apt*, и уменьшить размер образа.

  - `WARN - CIS-DI-0001: Create a user for the container`: 

      Контейнер запускается от имени пользователя *root*, что может быть небезопасно.

      Рекомендуется создать отдельного пользователя для выполнения задач в контейнере, чтобы повысить безопасность.

  - `INFO - CIS-DI-0005: Enable Content trust for Docker`

      Рекомендуется включить доверие к контенту (*Content Trust*) для **Docker**, что помогает удостовериться, что образы загружаются из надежных источников. 

  - `INFO - CIS-DI-0006: Add HEALTHCHECK instruction to the container image`

      Образ не содержит инструкции `HEALTHCHECK`, что позволяет **Docker** проверять состояние запущенных контейнеров.

  - `INFO - CIS-DI-0008: Confirm safety of setuid/setgid files`

      В образе найдены файлы с установленными флагами *setuid* и *setgid*, которые могут представлять потенциальные риски безопасности. Эти флаги — это специальные атрибуты файлов в Unix-подобных операционных системах, которые позволяют программам запускаться с правами владельца файла (для *setuid*) или с правами группы файла (для *setgid*).

### Исправляем образ так, чтобы при проверке через **dockle** не было ошибок и предупреждений

Исправляем *Dockerfile*:

![Dockerfile](../../images/part5/3.png)

Добавили следующие строки:

  - `USER root`

    Устанавливает пользователя root для последующих команд. Это необходимо для установки пакетов и выполнения действий, требующих прав администратора.

  - `RUN rm -rf /var/lib/apt/lists/*`

    Удаляет все списки пакетов, чтобы сэкономить место в образе.

  - `RUN useradd -m hello_user`

    Создает нового пользователя с именем *hello_user* и создает домашнюю директорию для него.

  - `RUN chown -R hello_user:hello_user /etc/nginx/nginx.conf /var/cache/nginx /home`

    Изменяет владельца указанных файлов и директорий на *hello_user*.

  - `RUN touch /var/run/nginx.pid`

    Создает файл для хранения **PID** процесса **NGINX**.
    
    > Файл **PID** используется для хранения идентификатора процесса, который управляет службой **NGINX**. Это позволяет другим процессам или скриптам легко находить и управлять этим процессом, например, завершать его или перезапускать. Если **NGINX** работает, он записывает свой **PID** в этот файл, что упрощает управление процессом и обеспечивает корректную работу системы.

  - `RUN chown -R hello_user:hello_user /var/run/nginx.pid`

    Изменяет владельца файла *nginx.pid* на *hello_user*.

  - `RUN chmod u-s /usr/bin/gpasswd /usr/bin/chsh /usr/bin/chfn /usr/bin/expiry /usr/bin/passwd /sbin/unix_chkpwd /usr/bin/chage /usr/bin/wall /bin/umount /bin/mount /usr/bin/newgrp /bin/su`

    Убирает **SUID**-биты у указанных файлов для повышения безопасности контейнера, чтобы уменьшить риск потенциальных атак.

    > **SUID (Set User ID)** — это бит, который позволяет запускать файл с правами владельца этого файла. Это значит, что программа будет выполняться с теми правами, которые имеет её владелец, а не тот пользователь, который её запустил.
    >
    > Убирая **SUID-биты** у определённых файлов, вы снижаете риск потенциальных атак. Если у злоумышленника есть возможность эксплуатировать уязвимость в программе, имеющей **SUID-бит**, он может получить доступ к привилегиям владельца (например, root), что может привести к компрометации системы. Убирание **SUID-битов** помогает предотвратить такие сценарии и повышает общую безопасность контейнера.

  - `USER hello_user`

    Меняет пользователя на *hello_user* для последующих команд.

  - `HEALTHCHECK CMD curl --fail http://localhost:81/ || exit 1`

    Добавляет проверку состояния контейнера. Если запрос к URL *localhost:81/* завершится неудачей, контейнер будет считаться нездоровым.
