## Part 1. Готовый докер

> English version: [../eng/Part1.md](Part1.md)

### Берём официальный докер-образ с **nginx** и выкачиваем его при помощи `docker pull`

**Конечно, чтобы работала команда `docker pull`, нужно сначала установить Docker.**

Итак, вызов команды docker pull:

![docker pull](pics/part1/1.png)

### Проверяем наличие докер-образа через `docker images`

Чтобы увидеть, что образ был успешно скачан, выполняем команду `docker images`, которая покажет список всех локально доступных образов. Найдём в списке **nginx**. В нашем случае это несложно, т.к. у нас всего один образ.

![docker images](pics/part1/2.png)

### Запускаем докер-образ через `docker run -d [image_id|repository]`

> - `-d` означает, что контейнер будет запущен в фоновом режиме (detached mode).>#
>
> - `--name <name>` задаёт имя контейнеру, чтобы его было легче идентифицировать. Если не указать `--name`, Docker создаст прикольное случайное имя для контейнера.

![docker run -d](pics/part1/3.png)

Мы получили код — это **идентификатор (ID)** контейнера. Он уникален для каждого запущенного контейнера и позволяет Docker отслеживать его состояние и управлять им.

### Проверяем, что образ запустился через `docker ps`

> Команда `docker ps` показывает список всех запущенных контейнеров.

![docker ps 1](pics/part1/4.png)

Как видим, наш контейнер *my-nginx* работает.

### Смотрим информацию о контейнере через `docker inspect [container_id|container_name]`

![docker inspect](pics/part1/5.png)

Открывается огромный страшный файл *JSON*.

### По выводу команды опредляем размер контейнера, список замапленных портов и ip контейнера

 - **Замапленные порты**: 80

![порты](pics/part1/6_1.png)

 - **IP контейнера**: 172.17.0.2

![IP](pics/part1/6_2.png)

 - **Размер контейнера**: 1.09kB

    Так как контейнер пустой, **Docker** не показывает его размер (*SizeRootFs*).

    ![размер](pics/part1/6_3.png)

    Однако благодаря команде `docker ps -s` мы можем узнать его размер (а также подтвердить замапленные порты).

    ![размер](pics/part1/6_4.png)

### Останавливаем докер контейнер через `docker stop [container_id|container_name]` и проверяем, что он остановился

![docker stop](pics/part1/7.png)

Как видим, список запущенных контейнеров пуст, значит, наш контейнер остановился.

### Запускаем докер с портами 80 и 443 в контейнере, замапленными на такие же порты на локальной машине, через команду `run`

> Назвать новый контейнер именем старого (в нашем случае *my-nginx*) не получится, т.к. он всё ещё существует, даже если остановлен. В данном случае мы просто позволили **Docker**'у выбрать случайное имя для нового контейнера.

> **-p 80:80 и -p 443:443** — это перенаправление портов. Первый порт — порт на локальной машине, а второй — порт внутри контейнера.

![run](pics/part1/8.png)

### Проверяем, что в браузере по адресу `localhost:80` доступна стартовая страница `nginx`

![localhost:80](pics/part1/9.png)

### Перезапускаем докер контейнер через `docker restart [container_id|container_name]` и проверяем, что контейнер запустился

Обратите внимание на столбец *STATUS*.

![docker restart](pics/part1/10.png)
